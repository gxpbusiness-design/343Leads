Create a Google Cloud Project
Every Google API lives inside a "project." Think of it like a container that holds your credentials and settings. You only need to do this once.
1
Go to console.cloud.google.com and sign in with your Google account.
2
At the top of the page, click the project selector dropdown (it usually shows "Select a project" or an existing project name).
3
In the popup that appears, click New Project in the top-right corner.
4
Give your project a name — something like 343Leads works great. The name is just for your reference.
5
Click Create and wait a few seconds for it to provision. You'll be taken into the project dashboard.
Tip: Make sure the new project is selected in the top dropdown before continuing to the next step — it's easy to accidentally be inside a different project.
Enable the Gmail API
By default, no APIs are turned on. You need to explicitly enable Gmail so your credentials can access it.
1
In the left sidebar, click APIs & Services → then Library.
2
In the search bar, type Gmail API and press Enter.
3
Click on the Gmail API result (it should be the first one, published by Google).
4
Click the blue Enable button. The page will reload and show you the Gmail API overview — that means it worked.
You may also want to enable Google People API the same way if you plan to sync contacts later.
Create Your OAuth Credentials
This is where you get the Client ID that you paste into 343Leads. OAuth lets the app access your Gmail without ever seeing your password.
1
Go to APIs & Services → Credentials in the left sidebar.
2
Click + Create Credentials at the top, then choose OAuth client ID.
3
If prompted to configure the consent screen first, click Configure Consent Screen. Choose External, fill in the app name (343Leads), add your email as support email, then click through Next until done. Then go back to Credentials → Create Credentials → OAuth client ID.
4
Set Application type to Web application.
5
Under Authorized JavaScript origins, click + Add URI. Add http://localhost if opening 343Leads as a local file, or your hosted domain like https://yourdomain.com. Note: Google does not allow file:// origins — if you're running locally, serve it with a simple local server or use VS Code Live Server.
6
Click Create. A popup will show your credentials. Copy the Client ID — it looks like 764…….apps.googleusercontent.com.
7
Paste that Client ID into the field on the Emails setup page and click Connect with Google. That's it!
You only need the Client ID — not the Client Secret. The secret is for server-side apps. 343Leads runs entirely in your browser so it only needs the ID.
